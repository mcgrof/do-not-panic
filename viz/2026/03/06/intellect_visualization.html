💡 Core Insight

DiLoCo turns the bandwidth problem upside-down. Instead of syncing gradients at every step (traditional data-parallel requires ~3.2 Tb/s InfiniBand), workers run 100 local AdamW steps independently, then sync pseudo-gradients compressed to int8 every ~38 minutes. The internet's 500 Mb/s is suddenly enough to train a 10B model across 5 countries.

PRIME framework components
⊞

ElasticDeviceMesh

Novel abstraction managing two-tier process groups: FSDP local groups for fast intra-node NVLink communication, and fault-tolerant global groups for internet-scale DiLoCo sync. Nodes can join or leave without killing the run.

global_rank ↔ local_rank hybrid topology
⬡

Int8 Ring-All-Reduce

Custom C++ multithreaded uint8 quantization — 60× faster than PyTorch implementation. Quantizes pseudo-gradient deltas (not weights), making it robust to outlier accumulation. Pipelined to overlap quant/dequant with network I/O.

4× size reduction · fp32 accumulation
◈

Fault Tolerance

Heartbeat every 2 seconds. Nodes evicted after 6s silence. Graceful exits send a "deathrattle" for immediate removal. Peer-to-peer checkpoint transfers let new nodes sync without halting training — P2P download completes in 30–60 min.

heartbeat_interval=2s · timeout=6s
⊛

Ring Topology Optimizer

Background process continuously measures bandwidth between all node pairs. Solves a max-bottleneck Hamiltonian cycle problem (variant of TSP) to find the ring order that maximizes minimum-bandwidth along the all-reduce ring.

max_C min_(u,v)∈C w(u,v)

💡 Core Insight

RL is inherently more asynchronous than pre-training. Rollout generation, policy training, and weight broadcasting can all proceed simultaneously with no blocking. INTELLECT-2 exploits this: inference workers (consumer GPUs!) generate reasoning traces while GRPO trainers update the policy asynchronously — zero communication overhead.

// INTELLECT-2 distributed RL pipeline
① Inference Workers
Consumer GPUs (3090s, A100s, MI300X, H100s) generate reasoning traces using vLLM in bf16. Samples questions deterministically via seed = node_addr · step + n_submissions to prevent cherry-picking. TOPLOC proofs built asynchronously on CPU with <1% overhead.
vLLM · bf16
② TOPLOC Validators
Locality-sensitive hashing scheme verifies inference integrity without redoing full inference. Detects quantized/malicious model usage, wrong sampling distributions, premature EOS termination. Validators run 100× faster than generation — cheap to operate.
LSH · 100×
③ GRPO Training Workers
Centralized trusted nodes run GRPO with PyTorch FSDP2 (ZeRO-3 style). Two-sided clipping prevents reward hacking. Computes log-probs using the current policy at optimization time (async-safe). Auxiliary KL + entropy losses.
FSDP2 · GRPO
④ SHARDCAST Weight CDN
HTTP tree-topology network (like a CDN) propagates updated policy weights to all inference workers. Checkpoints sharded and streamed in pipeline — workers begin downloading before the full checkpoint is ready. SHA-256 integrity check on assembly. Last 5 versions kept on relay servers.
CDN · SHA-256
TOPLOC — trustless inference verification
🔐

Computation Check

Worker generates a cryptographic commitment to final hidden states (top-k indices + polynomial encoding) every 32 tokens. Validator reconstructs via prefill and checks consistency. Works across non-deterministic GPU hardware and different tensor-parallel configs.

📊

Sampling Check

Correct sampling from logits produces an exponential distribution with mode at 1. If a smaller model is used for token generation (only using large model for prefill to pass TOPLOC), a telltale bimodal distribution at 0 and 1 is detected.

✓

Sanity Checks

EOS token probability must exceed 0.1. Rewards and advantages within predefined bounds. Parquet schema validation. Fixed deterministic sampling via seed — validator can verify the exact samples each worker should have used.

GRPO training recipe innovations
⚖

Two-Sided GRPO Clipping

Standard GRPO clips only the upper bound of policy ratios. INTELLECT-2 adds a lower-bound clip too — preventing the model from collapsing to near-zero probability on correct answers when an advantage signal is noisy. Critical for training stability at scale.

clip(r, 1-ε, 1+ε) → clip(r, ε_low, 1+ε_high)
⊘

Offline + Online Data Filtering

Offline: Remove training prompts where the base model already scores perfectly (no learning signal) or never scores (too hard). Online: During training, filter out rollout batches with zero variance in rewards — these carry no gradient signal and waste compute.

filter: var(rewards) == 0 → discard batch
📊 Result

INTELLECT-2 improves upon QwQ-32B (the state-of-the-art 32B reasoning model) across math and coding benchmarks, demonstrating that globally distributed asynchronous RL can successfully train reasoning capabilities — without any centralized GPU cluster.

// The decentralized training arc — 2024 → 2025
Step 1 — Solve bandwidth (INTELLECT-1)
DiLoCo proves that 100-step local training + infrequent pseudo-gradient sync can match dense gradient communication. int8 compression + topology-aware rings squeeze 400× bandwidth reduction out of standard internet connections. Result: 10B model, 1T tokens, 30 contributors.
Dec 2024
Step 2 — Go fully async (INTELLECT-2)
Recognize that RL's rollout→train→update cycle is naturally decoupled. Asynchronous GRPO eliminates the remaining sync bottleneck. TOPLOC makes trustless permissionless compute viable. SHARDCAST delivers weights to thousands of heterogeneous inference workers. Result: 32B reasoning model, beats SoTA.
May 2025
Step 3 — What's next?
PCCL (Prime Collective Communications Library) — a TCP-based protocol purpose-built for internet collective communication with dynamic membership, shared state sync, and bandwidth-aware topology. The datacenter network stack is finally being redesigned from scratch for the open internet.
